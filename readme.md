# 김밥 스터디 - GitHub 사용법

처음이라 어렵게 느껴질 수 있는데, 아래 순서대로만 따라하면 돼요. 천천히 같이 해봅시다.

---

## 1. 준비 (처음 한 번만)

### 1-1. 저장소(repository) 받아오기 (clone)
내가 올린 문제를 내 컴퓨터로 가져오는 단계예요.

1. 아래 깃허브 주소가 우리 저장소예요:
   👉 **https://github.com/01022445337a-alt/test1.git**
2. 내 컴퓨터에서 코드를 저장하고 싶은 폴더로 이동한 뒤, 그 폴더에서 **터미널(CMD / PowerShell / Git Bash)** 을 열어요.
3. 아래 명령어 입력:

```bash
git clone https://github.com/01022445337a-alt/test1.git
```

→ 그러면 `test1` 이라는 폴더가 하나 생겨요. 그 폴더 안으로 들어가세요.

```bash
cd test1
```

---

## 2. 문제 풀기

내가 `problem01.py`, `problem02.py` 이런 식으로 문제를 올릴 거예요.

1. 문제 파일을 열어서 안에 있는 설명을 읽어요.
2. **같은 폴더 안에** 본인 답안을 작성하면 돼요.
   - 파일을 직접 수정해서 풀어도 되고,
   - 본인 이름 폴더를 만들어서 그 안에 풀어도 좋아요. (예: `홍길동/problem01.py`)
3. 코드를 다 짰으면 저장(Ctrl + S)!

---

## 3. 깃허브에 올리기 (이게 핵심!)

푼 코드를 깃허브로 올리는 방법은 **딱 3줄** 이에요.

터미널에서 저장소 폴더(`test1`) 안에 있는지 먼저 확인하세요.

### 3-1. 변경사항 등록하기
```bash
git add .
```
> `.` 은 "변경된 모든 파일"이라는 뜻이에요. 내가 추가하거나 수정한 파일들을 "올릴 준비" 시키는 단계.

### 3-2. 메시지 남기면서 저장하기 (commit)
```bash
git commit -m "메시지내용"
```
> `-m` 뒤 따옴표 안에는 **무슨 작업을 했는지** 짧게 써주면 돼요.

예시:
```bash
git commit -m "problem01 풀이"
git commit -m "problem02 구구단 문제 해결"
git commit -m "홍길동 problem01 제출"
```

### 3-3. 깃허브로 보내기 (push)
```bash
git push origin master
```
> 내 컴퓨터에 저장된 commit을 **깃허브 서버로 올리는** 단계.
> 이걸 해야지 다른 사람도 내가 푼 코드를 볼 수 있어요!

---

## 4. 한눈에 보는 전체 흐름

```bash
# 1) 폴더로 이동
cd test1

# 2) 문제 풀고 저장한 다음...

# 3) 깃허브에 올리기 3종 세트
git add .
git commit -m "problem01 풀이 완료"
git push origin master
```

✅ 이거 세 줄만 외워두면 끝!

---

## 5. 자주 묻는 질문

### Q. push 했는데 로그인 창이 떠요!
→ 깃허브 아이디 / 비밀번호(또는 토큰) 입력하면 돼요. 한 번 로그인하면 다음부턴 안 물어봐요.

### Q. `git push` 했는데 에러가 나요. (`rejected` 뜸)
→ 누가 먼저 push 했을 수 있어요. 아래 명령어로 최신 코드를 먼저 받아오세요:
```bash
git pull origin master
```
그 다음에 다시 `git push origin master` 하면 됩니다.

### Q. 내가 올린 게 잘 올라갔는지 어떻게 확인해요?
→ 깃허브 저장소 페이지를 새로고침하면 내가 올린 파일이랑 commit 메시지가 보여요!

### Q. 명령어 실행할 때 폴더 위치가 헷갈려요.
→ 터미널에 `pwd` (또는 윈도우 CMD에선 `cd`) 쳐보면 현재 위치가 나와요. 반드시 **저장소 폴더 안**에서 명령어를 쳐야 합니다.

## 6. 다음 단계 (나중에 배울 것들)

지금은 위 3줄(`add` → `commit` → `push`)만 익히면 충분해요!
나중에 익숙해지면 아래 내용도 같이 배워볼 거예요:
- 브랜치(branch) 만들고 옮기기
- Pull Request 보내기
- 충돌(conflict) 해결하기
